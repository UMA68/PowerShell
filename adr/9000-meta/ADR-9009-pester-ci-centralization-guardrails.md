
# ADR-9009: ADR-0009（Pester CI集約）に対する運用ガードレール

## Status
Accepted

## Context
本 ADR は、ADR-0009 の判断内容を変更するものではなく、その弱点を**運用ルール**で補強するためのメタ ADR である。

ADR-0009 は、Pester 5 系の正式実行環境を GitHub Actions に集約し、ローカルは補助的に Pester 4.10.1 を許容する判断を採用した。
この判断は、ローカル環境の SRP / GPO 制約を回避する点では有効だが、赤チーム観点では以下の破綻リスクがある。

- CI 依存によるフィードバック遅延
- ローカル 4 系と CI 5 系の差分による見逃し
- CI ランナー更新や環境差分による再現不能障害
- 「1回だけ検証」の証拠劣化
- 再評価条件の未運用による判断凍結

## Decision
ADR-0009 を維持する場合、次を必須運用とする。

1. CI 実行環境（OS イメージ / PowerShell / Pester）を明示し、変更検知時は再検証を実施する。
2. ローカル Pester 4.10.1 は補助確認と定義し、**合否の最終判定には使用しない**。
3. PR 作成時またはマージ前に、Pester 5 系の**最小スモークテスト**を必ず実行する。
4. 「1回のみ試行」の証拠（条件・ログ・失敗パターン）を GitHub 上に保全する。
5. 再評価トリガー（GPO / SRP 変更、PowerShell 更新、Pester 更新、CI 失敗率悪化）を定義し、発生時に ADR-0009 を再審する。
6. CI 障害時の暫定判定手順を定義し、運用停止を防ぐ。

## Consequences

### 👍
- ADR-0009 の弱点を運用ルールで補強できる
- 「CI 一本化」の判断が属人的な慣習ではなく、再現可能なプロセスになる
- 将来の環境改善時に、判断を再評価へ戻す経路が明確になる

### 👎
- 監視項目と運用手順が増え、短期的な管理コストが上がる
- PR 前のスモーク実行により、開発体験はわずかに重くなる

## Notes

### ローカル検証の一次証拠（2026-02）
- ローカル環境（PowerShell 7.5.4 / Windows）において、以下を確認した：
  - **Pester 5.0.0**：AllUsers インストールで Import / 実行が可能
  - **Pester 5.6.1**：AllUsers インストールで Import / 最小テストが成功
  - **Pester 5.7.1**：同条件下で問題が再現
- よって、ローカル実行不可の原因は **Pester 5 系全般ではなく、5.7.x 以降の変更点と環境要因の組み合わせ**に起因すると判断する。

### 再評価条件
- 本判断は以下のイベント発生時に再評価対象とする：
  - 組織の GPO / SRP 設定変更
  - PowerShell のメジャー更新
  - Pester のメジャー、または互換性に影響する更新
  - CI（GitHub Actions）における原因不明な失敗率の増加

- 「Pester 5.0.x / 5.6.1 を1回だけ試行した」検証は、当時の制約下での一次証拠として扱い、
  再評価時には **同条件で再試行する前提**とする。

# 暗号化文字列の復元ツール

鍵ファイルを使用して暗号化された文字列を復号し、元の平文を表示するPowerShellスクリプトです。

## 概要

このツールは、`Encryption.key` などの鍵ファイルを使用して暗号化された文字列を復号します。暗号化した文字列が何だったか忘れた場合の救済処置として使用します。

**復号結果は2つの方法で表示されます：**
1. **ターミナル画面** - コピー＆ペースト用（便利！）
2. **ポップアップダイアログ** - 確認用

### 主な機能

- **安全な復号処理** - AES暗号化による鍵ベースの復号
- **GUIインターフェイス** - 使いやすいフォーム入力（Enter/Escapeキー対応）
- **ターミナル出力** - 復号結果を簡単にコピー＆ペースト可能
- **二重起動防止** - 同時実行を防ぐ安全機構
- **自動クリーンアップ** - 処理後に機密情報をメモリから削除

## システム要件

### PowerShell
- **バージョン**: 5.1以上（推奨: 7.3.9以上）
- **実行ポリシー**: RemoteSigned または Unrestricted

### 前提条件
- 鍵ファイル（Encryption.key）が `Common/` フォルダーに存在すること
- 暗号化時と同じ鍵ファイルを使用する必要があります

## 使い方

### 基本的な使用方法

1. **鍵ファイルの配置**
   - 暗号化時に使用した `Encryption.key` を `Common/` フォルダーに配置

2. **スクリプトの実行**
   - 「暗号文字列の復号.ps1 - ショートカット」をダブルクリック
   - または PowerShell で直接実行:
     ```powershell
     .\Script\StringDecryption.ps1
     ```

3. **文字列の入力**
   - 表示されたフォームに暗号化文字列を入力
   - 「復号実行」ボタンをクリック（またはEnterキーを押す）
   - キャンセルする場合は「キャンセル」ボタンまたはEscapeキーを押す

4. **結果の確認**
   - **ターミナル画面**に復号結果が表示されます（区切り線で強調表示）
   - 同時に**ポップアップダイアログ**でも確認できます
   - ターミナルの復号結果を選択してコピー＆ペーストできます

### 出力例

```
復号に成功しました。

=============================
復号結果: パスワード123
=============================

※ 上記の復号結果をコピーしてご利用ください
```

## トラブルシューティング

### よくあるエラー

| エラーメッセージ | 原因 | 対応方法 |
|-------------|------|--------|
| "鍵ファイル「XXX」が見つかりません" | 鍵ファイルが存在しない | `Common/` フォルダーに正しい鍵ファイルを配置 |
| "InputGUI.ps1 が見つかりません" | スクリプトファイルが存在しない | `Script/` フォルダーの構成を確認 |
| "文字列の復号に失敗しました" | 暗号文字列または鍵ファイルが不正 | 正しい暗号文字列と鍵ファイルを使用 |
| "すでに実行中です" | 同じスクリプトが実行されている | 既存プロセスの完了を待つ |

### よくある問題

#### 1. 復号に失敗する

**症状**: "文字列の復号に失敗しました" のエラーが表示される

**原因**:
- 暗号化時と異なる鍵ファイルを使用している
- 暗号文字列が破損している
- 暗号文字列のコピー時に改行や空白が混入している

**対応**:
- 暗号化時と同じ鍵ファイルを使用しているか確認
- 暗号文字列を正確にコピーペーストする
- 前後の空白や改行を削除する

#### 2. フォームが表示されない

**症状**: スクリプトを実行してもフォームが表示されない

**原因**:
- InputGUI.ps1 が見つからない
- PowerShell のバージョンが古い

**対応**:
```powershell
# InputGUI.ps1 の存在確認
Test-Path ".\Script\InputGUI.ps1"

# PowerShell バージョン確認
$PSVersionTable.PSVersion
```

## セキュリティに関する注意

### 機密情報の取り扱い

- **画面表示**: 復号結果は画面に表示されます。作業時は周囲に注意してください
- **メモリクリーンアップ**: 処理終了時、自動的に機密情報をメモリから削除します
- **ログ出力なし**: 復号結果はログに記録されません（セキュリティ保護）

### 鍵ファイルの管理

- 鍵ファイルは厳重に管理してください
- 鍵ファイルを紛失すると復号できなくなります
- 鍵ファイルのバックアップを安全な場所に保管してください

### 使用上の注意

- 信頼できる環境でのみ実行してください
- 暗号文字列を第三者と共有しないでください
- 復号後は速やかに画面を閉じてください

## よくある質問（FAQ）

### Q1: 復号に失敗する原因は？

**A**: 最も多い原因は以下の3つです：
1. 暗号化時と異なる鍵ファイルを使用している
2. 暗号文字列が正確にコピーされていない（改行や空白の混入）
3. 暗号文字列自体が破損している

### Q2: 鍵ファイルを紛失した場合は？

**A**: 鍵ファイルを紛失すると復号できません。鍵ファイルは必ずバックアップを取り、安全な場所に保管してください。

### Q3: 複数の鍵ファイルを管理するには？

**A**: `-keyFileName` パラメーターで鍵ファイル名を指定できます：
```powershell
.\StringDecryption.ps1 -keyFileName "Project1.key"
.\StringDecryption.ps1 -keyFileName "Project2.key"
```

### Q4: 復号結果はログに残りますか？

**A**: いいえ。セキュリティ上の理由から、復号結果はログファイルに記録されません。結果はポップアップダイアログと実行中のターミナル画面でのみ表示されます。

### Q5: どのような暗号化方式を使用していますか？

**A**: AES（Advanced Encryption Standard）を使用しています。PowerShell の `ConvertFrom-SecureString` / `ConvertTo-SecureString` による標準的な暗号化です。

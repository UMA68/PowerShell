
# ADR-9009: ADR-0009（Pester CI集約）に対する運用ガードレール

## Status
Accepted

## Context
本ADRは、ADR-0009の判断内容を変更するものではなく、その弱点を**運用ルール**で補強するためのメタADRである。

ADR-0009は、Pester 5系の正式実行環境をGitHub Actionsに集約し、ローカルは補助的にPester 4.10.1を許容する判断を採用した。
この判断は、ローカル環境のSRP / GPO制約を回避する点では有効だが、赤チーム観点では以下の破綻リスクがある。

- CI依存によるフィードバック遅延
- ローカル4系とCI 5系の差分による見逃し
- CIランナー更新や環境差分による再現不能障害
- 「1回だけ検証」の証拠劣化
- 再評価条件の未運用による判断凍結

## Decision
ADR-0009を維持する場合、次を必須運用とする。

1. CI実行環境（OSイメージ / PowerShell / Pester）を明示し、変更検知時は再検証を実施する。
2. ローカルPester 4.10.1は補助確認と定義し、**合否の最終判定には使用しない**。
3. PR作成時またはマージ前に、Pester 5系の**最小スモークテスト**を必ず実行する。
4. 「1回のみ試行」の証拠（条件・ログ・失敗パターン）をGitHub上に保全する。
5. 再評価トリガー（GPO / SRP変更、PowerShell更新、Pester更新、CI失敗率悪化）を定義し、発生時にADR-0009を再審する。
6. CI障害時の暫定判定手順を定義し、運用停止を防ぐ。

## Consequences

### 👍
- ADR-0009の弱点を運用ルールで補強できる
- 「CI一本化」の判断が属人的な慣習ではなく、再現可能なプロセスになる
- 将来の環境改善時に、判断を再評価へ戻す経路が明確になる

### 👎
- 監視項目と運用手順が増え、短期的な管理コストが上がる
- PR前のスモーク実行により、開発体験はわずかに重くなる

## Notes

### ローカル検証の一次証拠（2026-02）
- ローカル環境（PowerShell 7.5.4 / Windows）において、以下を確認した：
  - **Pester 5.0.0**：AllUsersインストールでImport / 実行が可能
  - **Pester 5.6.1**：AllUsersインストールでImport / 最小テストが成功
  - **Pester 5.7.1**：同条件下で問題が再現
- よって、ローカル実行不可の原因は **Pester 5 系全般ではなく、5.7.x 以降の変更点と環境要因の組み合わせ**に起因すると判断する。

### 再評価条件
- 本判断は以下のイベント発生時に再評価対象とする：
  - 組織のGPO / SRP設定変更
  - PowerShellのメジャー更新
  - Pesterのメジャー、または互換性に影響する更新
  - CI（GitHub Actions）における原因不明な失敗率の増加

- 「Pester 5.0.x / 5.6.1を1回だけ試行した」検証は、当時の制約下での一次証拠として扱い、
  再評価時には **同条件で再試行する前提**とする。


# 暗号化文字列の作成スクリプト ReadMe

このスクリプトは、入力した文字列を指定した鍵ファイル（既定：Common\Encryption.key）で暗号化し、暗号化文字列をファイルとして出力します。パスワードなど平文で保存したくない情報の管理に便利です。

## 主な機能

- 鍵ファイルを使用した安全な暗号化処理
- 二重起動防止機構（すでに起動中の場合は警告ダイアログを表示）
- SecureStringを使用したセキュアな文字列処理
- 処理完了時の確認ダイアログ表示
- 処理後の自動変数クリーンアップ

---

## 使い方

1. PowerShellで本スクリプトを実行します。

   ```powershell
   ./Script/MakeEncryptedString.ps1
   ```

2. プロンプトにしたがって暗号化したい文字列を入力します。
3. 入力した文字列がターミナルに表示されます（確認用）。
4. 暗号化された文字列がターミナルに表示されます。
5. 出力するファイル名を入力します（このReadMeと同じディレクトリに保存されます）。
6. 完了ダイアログが表示され、暗号化文字列が指定ファイルに保存されます。

### サンプル入出力例

```powershell
./Script/MakeEncryptedString.ps1
```

```text
暗号化する文字列を入力してください: mySecretPassword
暗号化する文字列: mySecretPassword
暗号化した文字列: <暗号化済み文字列>
出力するファイル名を入力してください: secret.txt
[ダイアログ表示] 暗号化した文字列をファイル「secret.txt」に出力しました。
```

### 鍵ファイルの指定

デフォルトは`Common/Encryption.key`です。別の鍵ファイルを使いたい場合は、
`-keyFileName`パラメーターでファイル名を指定し、Commonフォルダーに配置してください。

```powershell
./Script/MakeEncryptedString.ps1 -keyFileName MyKey.bin
```

## 注意事項

- **二重起動**: スクリプトの二重起動は防止されています。すでに起動中の場合は警告ダイアログが表示されます。
- **セキュリティ**: 入力文字列はSecureStringとして処理され、処理後は自動的にメモリから削除されます。
- **ファイル名**: ファイル名や鍵ファイル名にパス区切り文字（/、\、:）は使えません。
- **鍵ファイル**: 鍵ファイルが存在しない場合はエラーとなります（「Common」フォルダーに配置してください）。
- **復号**: 暗号化文字列の復号には [`暗号化文字列の復元/Script/StringDecryption.ps1`](../暗号化文字列の復元/Script/StringDecryption.ps1) を利用してください。
- **鍵作成**: 鍵ファイルの作成は [`暗号化鍵の作成/Script/MakeEncrypted.ps1`](../暗号化鍵の作成/Script/MakeEncrypted.ps1) で行えます。
- **管理**: 鍵ファイルや暗号化ファイルは厳重に管理し、第三者に漏洩しないようご注意ください。
